Moniteur de responsabilité hivernale

MONITEUR DE RESPONSABILITÉ HIVERNALE : 2026–2027. Notre mandat principal consiste à vérifier si la Ville de Toronto respecte son obligation humanitaire : zéro personne sans place lorsque la température descend à −5 °C ou en dessous. Six hivers de données de l’Admission centrale montrent que la Ville n’a respecté cette norme que lors de 4,1 % des nuits admissibles. La surveillance quotidienne reprendra à l’ouverture de la saison froide 2026–2027 en novembre.

function fmtDate(s) {
  const [y, m, d] = s.split("-").map(Number)
  return new Date(y, m - 1, d).toLocaleDateString("fr-CA", {
    year: "numeric", month: "long", day: "numeric"
  })
}

  const rows = latestRows

  const expectedKeys = [

  const presentKeys = new Set(rows.map(r => r.key))
  const missingKeys = expectedKeys.filter(k => !presentKeys.has(k))
  checks.push({
    name: "Toutes les lignes attendues présentes",
    pass: missingKeys.length === 0,
    detail: missingKeys.length > 0 ? `Manquantes : ${missingKeys.join(", ")}` : null
  })

  const allShelter = getVal(rows, "all_shelter")
  const compSum = ["fam_total","sng_hotel","singles_total","allied_total","temp_resp","hotels","iso"]
    .reduce((s, k) => s + (getVal(rows, k) ?? 0), 0)
  checks.push({
    name: "Le total Tous les programmes de refuge correspond aux composantes",
    pass: allShelter === compSum,
    detail: allShelter !== compSum
      ? `all_shelter.ind=${allShelter}, somme des composantes=${compSum}, diff=${allShelter - compSum}`
      : null
  })

  const singlesTotal = getVal(rows, "singles_total")
  const emergTotal = getVal(rows, "emerg_total")
  const transTotal = getVal(rows, "trans_total")
  checks.push({
    name: "Total secteurs individus : urgence + transition = total individus",
    pass: singlesTotal === (emergTotal + transTotal),
    detail: singlesTotal !== (emergTotal + transTotal)
      ? `individus=${singlesTotal}, urgence+transition=${emergTotal + transTotal}`
      : null
  })

  const capRows = rows.filter(r => r.cap !== null && r.cap > 0)
  const arithFails = capRows.filter(r => Math.abs(r.occ + r.unocc - r.cap) > 0.5)
  checks.push({
    name: "Arithmétique interne : occupé + inoccupé = capacité réelle",
    pass: arithFails.length === 0,
    detail: arithFails.length > 0
      ? arithFails.map(r => `${r.key}: ${r.occ}+${r.unocc}≠${r.cap}`).join("; ")
      : null
  })

  return checks
}

function offlineCell(r) {
  if (r.cap === null || r.cap === undefined) return `<td></td>`
  if (r.cap_funding === null || r.cap_funding === undefined) return `<td></td>`
  const offline = r.cap_funding - r.cap
  if (offline <= 0) {
    return `<td class="bq-num">${offline.toLocaleString("fr-CA")}</td>`
  }
  return `<td class="bq-num bq-danger">${offline.toLocaleString("fr-CA")}</td>`
}

Le sous-total Tous les programmes de refuge et toutes les lignes de détail sont dérivés de l’exportation des données ouvertes CKAN. Les Programmes de passerelle et de triage sont suivis dans le système opérationnel de la Ville mais ne sont pas publiés dans les données ouvertes ; BonQuery collecte ce chiffre quotidiennement depuis la page du recensement des refuges de la Ville et l’affiche lorsqu’il est disponible. Le Total des personnes hébergées égale Tous les programmes de refuge + Passerelle et triage lorsque les deux sont disponibles. Certains chiffres de capacité et de lits inoccupés peuvent différer légèrement du tableau de bord en direct de la Ville en raison de différences dans la déclaration des lits hors service entre l’exportation CKAN et le système opérationnel de la Ville. NoteDétection historique de pics d’occupation

Ces graphiques de détection de pics signalent les jours où le réseau de refuges de Toronto a hébergé beaucoup plus de personnes que ne l’auraient prédit son pattern saisonnier et sa tendance à long terme — et pas seulement des journées simplement achalandées. Les pics sont identifiés en supprimant deux forces prévisibles (saisonnalité et tendance à long terme) par une décomposition saisonnelle-tendance par LOESS (STL) et en mesurant ce qui reste.

La bande ambre signale les anomalies modérées (2 MAD-ÉT au-dessus de la valeur attendue), la bande rouge les anomalies prononcées (3 MAD-ÉT). L’analyse compte les personnes hébergées plutôt que le taux d’occupation, et les deux seuils sont calibrés sur le comportement propre du système au cours des deux années précédentes. Elle est rétrospective et descriptive, non prédictive. La méthodologie complète et les mises en garde figurent sur la page liée.
