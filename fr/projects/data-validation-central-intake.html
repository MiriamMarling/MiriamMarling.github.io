Validation — Admission centrale

Valeurs calculées par BonQuery c. chiffres publiés par la Ville de Toronto

Validation des statistiques d’appels à l’Admission centrale de BonQuery par rapport au tableau des demandes d’orientation dans les refuges publié par la Ville de Toronto, plus une estimation mensuelle de la population Passerelle et triage absente de l’exportation CKAN.

Auteur(-trice)

Miriam Marling

Cette page valide les moyennes mensuelles calculées par BonQuery pour les trois indicateurs de l’Admission centrale par rapport au tableau Demandes d’orientation dans les refuges publié par la Ville de Toronto (instantané de référence : 2026-05-25, couvrant avril 2024 – avril 2026). La page quantifie également l’écart mensuel entre l’occupation nocturne de BonQuery basée sur CKAN et le chiffre publié par la Ville — cet écart représente les programmes Passerelle et triage suivis dans le système opérationnel de la Ville mais non publiés dans les données ouvertes.

MONTH_ABB = ["","janv.","févr.","mars","avr.","mai","juin",
             "juill.","août","sept.","oct.","nov.","déc."]

bqIndex = {
  const idx = new Map()
  ci.monthly.forEach(r => idx.set(r.year * 100 + r.month, r))
  return idx
}

passRef  = (bq, city) => bq != null && Math.abs(bq - city) <= 0.10
passUnm  = (bq, city) => bq != null && Math.abs(bq - city) <= 0.10
passHdl  = (bq, city) => bq != null && Math.abs(bq - city) <= 1.0

mark = (pass, bqVal) =>
  bqVal == null
    ? `<span style="color:var(--bq-fg-disabled)">—</span>`
    : pass
      ? `<span style="color:var(--bq-success);font-weight:700">✓</span>`
      : `<span style="color:var(--bq-danger);font-weight:700">✗</span>`

function valRow(c, i) {
  const bq  = bqIndex.get(c.year * 100 + c.month)
  const br  = bq ? bq.referred_mean  : null
  const bu  = bq ? bq.unmatched_mean : null
  const bh  = bq ? bq.handled_mean   : null
  const pr  = passRef(br, c.referred)
  const pu  = passUnm(bu, c.unmatched)
  const ph  = passHdl(bh, c.handled)

  const bg  = i % 2 === 0 ? "var(--bq-bg)" : "var(--bq-bg-alt)"
  const td  = `padding:5px 9px;text-align:center;border-bottom:1px solid var(--bq-border)`
  const tdb = td + ";border-left:2px solid var(--bq-chart-blue)"

  return html`<tr style="background:${bg}">
    <td style="${td}">${c.year}</td>
    <td style="${td}">${MONTH_ABB[c.month]}</td>
    <td style="${tdb}">${br != null ? br.toFixed(1) : "—"}</td>
    <td style="${td}">${c.referred.toFixed(1)}</td>
    <td style="${td}">${mark(pr, br)}</td>
    <td style="${tdb}">${bu != null ? bu.toFixed(1) : "—"}</td>
    <td style="${td}">${c.unmatched.toFixed(1)}</td>
    <td style="${td}">${mark(pu, bu)}</td>
    <td style="${tdb}">${bh != null ? Math.round(bh).toLocaleString("fr-CA") : "—"}</td>
    <td style="${td}">${c.handled.toLocaleString("fr-CA")}</td>
    <td style="${td}">${mark(ph, bh)}</td>
  </tr>`
}

Échecs

{
  const failures = cityRef.flatMap(c => {
    const bq = bqIndex.get(c.year * 100 + c.month)
    const br = bq ? bq.referred_mean  : null
    const bu = bq ? bq.unmatched_mean : null
    const bh = bq ? bq.handled_mean   : null
    const out = []
    if (br != null && !passRef(br, c.referred))
      out.push(`${MONTH_ABB[c.month]} ${c.year} — Orientés : BonQuery ${br.toFixed(1)} c. Ville ${c.referred.toFixed(1)} (diff. ${(br - c.referred).toFixed(2)})`)
    if (bu != null && !passUnm(bu, c.unmatched))
      out.push(`${MONTH_ABB[c.month]} ${c.year} — Sans jumelage : BonQuery ${bu.toFixed(1)} c. Ville ${c.unmatched.toFixed(1)} (diff. ${(bu - c.unmatched).toFixed(2)})`)
    if (bh != null && !passHdl(bh, c.handled))
      out.push(`${MONTH_ABB[c.month]} ${c.year} — Traités : BonQuery ${Math.round(bh).toLocaleString("fr-CA")} c. Ville ${c.handled.toLocaleString("fr-CA")} (diff. ${(bh - c.handled).toFixed(1)})`)
    return out
  })

  if (failures.length === 0)
    return html`<p style="color:var(--bq-success)">Toutes les cellules sont conformes.</p>`

  return html`<ul style="font-size:0.9em">
    ${failures.map(f => html`<li>${f}</li>`)}
  </ul>`
}

Mai 2024 : Total des appels traités. La moyenne mensuelle calculée par BonQuery (976,6 appels/jour) est inférieure de 16,4 appels/jour au chiffre publié par la Ville (993). L’extrait BonQuery couvre les 31 jours sans zéro ni valeur nulle, ce qui écarte les données manquantes comme cause possible. Cela indique une différence de périmètre dans l’extrait CKAN pour ce mois-là — peut-être un programme ou un type d’appel que la Ville inclut dans son chiffre publié mais qui est absent de la ressource de données ouvertes. Tous les autres mois concordent à ±0,5 près, ce qui est cohérent avec l’arrondissement à l’entier par la Ville.

Nov 2024 : Appelants sans jumelage. BonQuery 191,4 c. Ville 191,6 (diff. −0,2). Juste à l’extérieur de la tolérance de ±0,1 ; probablement une différence d’arrondissement dans la façon dont la Ville calcule sa moyenne mensuelle publiée.

Estimation de la population Passerelle et triage (non disponible dans l’exportation CKAN)

Le chiffre d’occupation nocturne publié par la Ville inclut les programmes suivis dans son système opérationnel qui ne sont pas publiés dans les données ouvertes (Programmes Passerelle et triage). L’occupation de BonQuery est calculée uniquement à partir de l’exportation CKAN. L’écart ci-dessous — Ville moins BonQuery — est une estimation de la population Passerelle et triage pour chaque mois. Il ne s’agit pas d’un échec de validation ; c’est une différence structurelle attendue entre les deux sources de données.

Les écarts plus importants de la fin 2024 au début 2025 coïncident avec le pic de la réponse d’urgence aux refuges hivernaux, lorsque les programmes Passerelle et triage fonctionnent probablement à plus grande capacité. Les écarts plus faibles durant les mois d’été concordent avec une activité réduite de la réponse d’urgence.

Source des données

Toutes les données proviennent du jeu de données Appels à l’Admission centrale et du jeu de données Utilisation quotidienne des refuges et services d’hébergement de nuit, occupation et capacité de la Ville de Toronto, publiés sur le portail des données ouvertes de la Ville. Les valeurs de référence de la Ville proviennent de la page Demandes d’orientation dans les refuges, instantané du 2026-05-25.
