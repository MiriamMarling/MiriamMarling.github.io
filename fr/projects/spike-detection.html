Détection de pics d’occupation

Réseau de refuges de Toronto — jours anormalement au-dessus des prévisions saisonnières et de tendance

Détection rétrospective de pics pour l’occupation quotidienne des refuges de Toronto — identification des jours nettement au-dessus des prévisions saisonnières et de tendance par décomposition STL et seuils robustes.

Auteur(-trice)

Miriam Marling

Un pic est un jour où le réseau de refuges a hébergé nettement plus de personnes que ce que la saison et la tendance auraient laissé prévoir — et non simplement une journée chargée. Le réseau suit des schémas bien établis : l’occupation augmente en hiver lorsque le froid stimule la demande, et diminue en été. Il porte également une tendance à long terme qui reflète les changements structurels du marché du logement et des politiques d’accueil des réfugiés. Cette analyse utilise la décomposition STL pour éliminer mathématiquement ces deux forces, ne conservant que la partie inexpliquée. Un pic est un jour où ce résidu inexpliqué est exceptionnellement élevé. Les graphiques de cette page appliquent la détection de pics aux données quotidiennes d’occupation et de capacité des services d’hébergement de nuit de la Ville — une analyse qui, à notre connaissance, n’avait pas été publiée auparavant.

Cette approche mesure les personnes hébergées (SERVICE_USER_COUNT dans les données ouvertes de la Ville), et non le taux d’occupation. La distinction importe. Un pic de dénombrement signifie sans ambiguïté que plus de personnes étaient dans le réseau ce jour-là. Un pic de taux pourrait signifier plus de personnes, ou moins de lits disponibles — deux situations très différentes aux implications politiques distinctes. Le dénombrement est le bon signal pour détecter les événements de stress.

Il s’agit d’une analyse rétrospective et descriptive. Elle identifie les jours statistiquement anormaux compte tenu du comportement du réseau au cours des deux années précédentes — elle ne prédit pas les pics futurs. Les seuils sont calibrés sur le comportement récent du réseau lui-même, et non sur une norme externe.

La page affiche deux bandes seuil. La bande ambre (2 MAD-ÉT, soit deux écarts absolus médians en écart-type au-dessus de la valeur attendue) signale davantage de jours et capte les anomalies plus subtiles — utile pour constituer un dossier complet. La bande rouge (3 MAD-ÉT) ne signale que les pics prononcés peu susceptibles de résulter du bruit quotidien habituel. Les deux seuils sont calibrés sur les deux dernières années de comportement du réseau, si bien que le seuil monte ou descend selon que le réseau devient plus ou moins volatil.

data = FileAttachment("../data/spike_detection.json").json()
spikeData = data.map(d => ({
  ...d,
  dateObj: new Date(d.date + "T00:00:00")
}))
sectorRanked = ["men","women","mixed_adult","youth","families"].map(sec => {
  const rows = spikeData.filter(d => d.series === sec)
  const sp   = rows.filter(d => d.spike_flag !== "none").length
  const sp3  = rows.filter(d => d.spike_flag === "3mad").length
  return { key: sec, label: S.sectorLabels[sec], spikes: sp, spikes3: sp3 }
}).sort((a, b) => b.spikes - a.spikes)

topSec    = sectorRanked[0]
secondSec = sectorRanked[1]

ratioPhrase = (secondSec.spikes > 0 && (topSec.spikes / secondSec.spikes) >= 1.8)
  ? S.ratioHigher
  : S.ratioSame
windowOptions = [
  {label: S.windowLabels[0], months: 1},
  {label: S.windowLabels[1], months: 3},
  {label: S.windowLabels[2], months: 6},
  {label: S.windowLabels[3], months: 12},
  {label: S.windowLabels[4], months: null}
]
viewof selectedWindow = Inputs.select(windowOptions, {
  label: S.windowInputLabel,
  format: d => d.label,
  value: windowOptions[4]
})
viewof showBands = Inputs.checkbox([S.band2, S.band3], {
  label: S.bandsLabel,
  value: [S.band2, S.band3]
})
windowCutoff = selectedWindow.months === null
  ? new Date("2021-01-01")
  : (() => {
      const d = new Date()
      d.setMonth(d.getMonth() - selectedWindow.months)
      return d
    })()

systemData = spikeData
  .filter(d => d.series === "system" && d.dateObj >= windowCutoff)
  .sort((a, b) => a.dateObj - b.dateObj)

heroSectorData = spikeData
  .filter(d => d.series === topSec.key && d.dateObj >= windowCutoff)
  .sort((a, b) => a.dateObj - b.dateObj)

sectorOrder  = ["men", "women", "mixed_adult", "youth", "families"].filter(s => s !== topSec.key)
sectorLabels = S.sectorLabels
xAxis = (() => {
  if (!systemData.length) return {type: "time", label: null}
  const spanDays   = (systemData[systemData.length-1].dateObj - systemData[0].dateObj) / 86400000
  const spanMonths = spanDays / 30.44
  return {
    type: "time",
    label: null,
    ticks: spanMonths <= 3 ? d3.utcDay.every(7) : spanMonths <= 18 ? d3.utcMonth.every(1) : 8,
    tickFormat: d => {
      const m = d.getMonth(), y = d.getFullYear()
      if (spanMonths <= 18) return m === 0 ? `${S.months[m]}-${y}` : S.months[m]
      return m === 0 ? String(y) : m === 6 ? S.julLabel(y) : ""
    }
  }
})()

fmtN = S.fmtN

      }
    }))

html`<h3>${S.heroHeading(topSec.label)}</h3>`
heroStats = {
  const sectors = ["men","women","mixed_adult","youth","families"]
  const topRows = spikeData.filter(d => d.series === topSec.key)
  const topSpikes   = topRows.filter(d => d.spike_flag !== "none").length
  const topSevere   = topRows.filter(d => d.spike_flag === "3mad").length
  const totalSpikes = spikeData.filter(d => sectors.includes(d.series) && d.spike_flag !== "none").length
  const pct = Math.round(topSpikes / totalSpikes * 100)
  const startYear = spikeData.reduce((mn, d) => Math.min(mn, +d.date.slice(0,4)), 9999)
  return {topSpikes, topSevere, totalSpikes, pct, startYear, topLabel: topSec.label}
}
S.heroBox(heroStats, ratioPhrase)
sectorStats = ["men", "youth", "families", "women", "mixed_adult"].map(sec => {
  const rows    = spikeData.filter(d => d.series === sec)
  const spikes  = rows.filter(d => d.spike_flag !== "none").length
  const spikes3 = rows.filter(d => d.spike_flag === "3mad").length
  return {
    label: S.sectorLabels[sec],
    spikes,
    spikes3,
    pctDays: spikes / rows.length * 100,
    isTop: sec === topSec.key
  }
}).sort((a, b) => b.spikes - a.spikes)
